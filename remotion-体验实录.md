# Remotion Skill 体验实录

> 2小时踩坑记录：从"丑哭"到"有点意思"

---

## 基本信息

- **体验时长**：2+ 小时
- **产品**：pdf2skills 宣传视频
- **工具**：Claude Code + remotion skill
- **官方仓库**：[remotion-dev/skills](https://github.com/remotion-dev/skills)（JonnyBurger 维护）
- **成本参考**：~$3 token/视频（Claude API）

---

## 过程线

### 第一轮：详细计划，效果丑哭

**输入**：完整的视频制作计划
- 6个场景分镜脚本（痛点→问题→方案→流程→案例→CTA）
- 项目结构设计
- 设计规范表格（颜色、字体、帧率）
- 背景音乐方案

**结果**：丑哭

**问题**：计划详细 ≠ 执行到位。抽象的设计规范（"科技极简黑白主色调"）无法转化为具体的视觉效果。

---

### 第二轮：加载最佳实践，仍然不行

**操作**：
```
这个视频做得太丑太丑太丑了！我要加载 remotion-best-practices
```

**CC 响应**：
- Searched for 3 patterns, read 3 files
- Read 4 files（最佳实践文档）
- "明白，我会用这些最佳实践重新设计"

**结果**：还是很差劲

---

### 第三轮：细颗粒度反馈，效果变好

**关键输入**：
```
我要开始细颗粒度地提建议了！

1. 整个视频是黑底的太丑了，你可以换为一个纯色的底版的，都会更加好看一点！
2. 你的字和整个视频篇幅的大小太不匹配了，字和符号都太小太小了
3. 需要配一下音乐也可以简单配上字符的注解，听不到声音
4. 内容的拆分做得还是很不错的
```

**结果**：效果变好，值得保存

---

## 核心洞察

### 什么有效，什么无效

| 无效输入 | 有效输入 |
|---------|---------|
| "科技极简、黑白主色调" | "黑底太丑，换纯色" |
| 设计规范表格 | "字太小太小了" |
| 项目结构设计 | "需要配音乐" |
| 分镜脚本描述 | 具体的视觉问题反馈 |

### 结论

**Remotion skill 需要的是「可执行的视觉指令」，不是「设计方案文档」。**

---

## 当前能力边界

### 做得好的
- 内容的拆分理解
- 元素的配对
- 基于反馈的迭代改进

### 做得不好的
- 审美（需要大大提升）
- 从抽象描述推断视觉效果
- 无法直接生成"可用"视频

### 整体评价

> 无法非常完整地从语义理解直接生成可用视频，但已经有惊艳的感觉了。

## 给后来者的建议

1. 不要期待"给个大纲就能出片"
2. 准备好迭代：第一版大概率不满意
3. 反馈要具体：说"字太小"比说"设计规范"有用
4. 渲染很慢，可以先录屏预览
5. 成本约 $3/视频，做好预算
