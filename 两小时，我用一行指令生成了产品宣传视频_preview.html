remotion skill 试用，一行指令生成产品宣传视频可行

先说结论：能用，但需要很强的输入修正能力和审美输入。

试了 Claude Code 的 remotion skill。

一行指令，AI 帮你生成视频。Remotion 创始人 JonnyBurger 亲自维护的官方 skill，各种传言说可以超过剪映。

听起来很香，试用有惊艳的感受，但还没有达到完全直接可用。直接可用就是爆发点了。

实际体验？折腾了两个多小时，踩了三个坑，到现在视频其实还没渲染导出。

我的输入

就一句话：

“用 remotion skill 帮我为这篇文章宣传的产品生成一个产品视频”

然后它自己输出了一整套东西：

  • • 6 个场景分镜脚本（痛点引入 → 问题阐述 → 解决方案 → 工作流程 → 案例展示 → CTA）
  • • 完整项目结构设计
  • • 设计规范表格（#0D0D0D 深黑背景、#00FF88 科技绿强调色、JetBrains Mono 字体、30fps、1080×1080）
  • • 背景音乐方案

这部分是惊艳的 —— 它真的理解了产品，自己拆解出了完整的视频结构。

第一版：丑哭

然后它开始生成代码、跑起来。

出来的东西，丑得我想砸键盘。黑漆漆一片，字小得像蚂蚁，完全不是我想的「科技极简」。

问题：它能规划，但执行出来的视觉效果跟规划是两回事。

第二版：加载最佳实践，还是丑

我喊话：

“这个视频做得太丑太丑太丑了！我要加载 remotion-best-practices”

它很听话，去读了一堆文档：

  • • Searched for 3 patterns, read 3 files
  • • Read 4 files
  • • “明白，我会用这些最佳实践重新设计”

结果？还是很差劲。

发现：框架的最佳实践解决的是代码规范，不是视觉审美。

第三版：细颗粒度反馈，效果变好

我换策略，开始给具体的视觉反馈：

我要开始细颗粒度地提建议了！

1. 整个视频是黑底的太丑了，换为一个纯色的底版
2. 字和符号太小太小了
3. 需要配音乐，加上字符注解
4. 内容的拆分做得还是很不错的

效果立刻变好。

核心发现：

没用的 有用的
“科技极简风格” “黑底太丑，换纯色”
设计规范表格 “字太小太小了”
分镜脚本描述 “加音乐”

它需要的是 可执行的视觉指令，不是设计方案文档。

渲染大坑

效果满意了，想保存下来。

50 秒视频，渲染 + 下载，接近一小时还没完。

我的曲线救国方案：为了展示这个效果，直接录屏预览界面。代价是音频丢了。

能力边界

做得好的：

  • • 内容拆分理解（一句话输入 → 完整分镜脚本）
  • • 元素配对
  • • 基于反馈快速迭代

做得不好的：

  • • 审美（默认出来的大概率不能用）
  • • 从抽象描述推断具体视觉

成本

有人算过，一个视频约 $3 token（Claude API）。

判断

整体评价：无法从语义理解直接生成可用视频，但已经有惊艳的感觉了，尤其是内容拆分和元素配对。审美需要大大提升。我估计这个在剪辑视频上会有更加惊艳的效果，剪辑视频是一个更加工程的事儿，而根据语义生成视频还是需要更强审美的。

现阶段定位：原型工具。不能「给个大纲就出片」，但迭代两三轮能出「够用」的版本。

下一步：试试更细颗粒度的图片输入，可能增强效果。另外据说用它剪辑视频效果更惊艳。
